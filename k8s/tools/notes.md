## kubectl
- official CLI for K8s
- used for interacting with the cluster
- kubectl uses REST API to interact with the cluster, which you could also use directly

## kubadm
- used to easily create a K8s cluster
- helps setup and make the cluster operational

## minikube
- tool to run a single-node K8s cluster locally
- suitable for development/testing

## Helm
- package manager for K8s
- can convert objects into reusable templates

## Kompose
- tool to convert docker-compose files into K8s objects
- ability to ship containers from compose to K8s

## Kustomize
- management tool K8s object configuration
- can convert objects into reusable templates, similar to Helm
